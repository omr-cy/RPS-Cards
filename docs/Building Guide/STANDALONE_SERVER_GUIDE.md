# دليل تشغيل أو استضافة السيرفر بشكل مستقل (Standalone Server)

مرحباً! لقد قمت بإنشاء طريقة لجعلك تستطيع رفع سيرفر اللعبة (Backend) بشكل مستقل تماماً على أي خدمة استضافة (مثل Render, Heroku أو VPS) بعيداً عن ملفات الواجهة (Frontend).

## ما هي المشكلة التي حدثت؟
التعليق الذي رأيته ("Connecting...") كان بسبب أن بيئة العمل المحلية أو AI Studio تستخدم مسار (Path) واحد لكلا من (تحديثات اللعبة الحية HMR الخاصة بواجهة React) و(أوامر لعبة البطاقات)، مما جعل الواجهة تحاول التحدث مع سيرفر التطوير الخاص بـ Vite بدلاً من سيرفر اللعب.

### الحل الجذري:
1. **تم تخصيص مسار خاص للعبة**: الآن اللعبة تتواصل على مسار محدد وهو `/game-socket` وهو ما يمنع أي تداخل مع أنظمة React.
2. **تم فصل ملف السيرفر**: يمكنك الآن تشغيل اللعبة محلياً بدون تداخل، وتم إعداد إمكانية استضافة السيرفر لوحده في مجلد `standalone-server`.

---

## كيف تستضيف السيرفر الأونلاين لوحده (VPS/Render/Host)؟

لقد قمت بإعداد كود خفيف ونظيف مخصص للباك إنـد فقط، يعمل بـ `Node.js` بدون `React` أو `Vite` لتسهيل رفعه، إذا كنت ترغب بتشغيله على سيرفر آخر، إتبع الخطوات:

1. انسخ ملفات `package.json` و `online_server.ts` إلى سيرفرك الخاص. (أو إذا أردت رفعهما بشكل منفصل).
2. على سيرفرك، قم بتثبيت الحزم الأساسية بإستخدام: `npm install ws express` فقط (لا حاجة للبقية).
3. في ملف `online_server.ts` الذي سترفعه، قم بإزالة أسطر الـ `Vite` في نهاية الملف:
```ts
// احذف هذا الكود من الملف الموجود على سيرفرك الخاص
if (process.env.NODE_ENV !== 'production') {
  const vite = await createViteServer({ ... });
  app.use(vite.middlewares);
}
```
4. قم بتشغيله على السيرفر الخاص بـ: `npx tsx online_server.ts`.

---

## كيف تربط التطبيق (Frontend) بالسيرفر المستقل الخاص بك؟

إذا قمت بتشغيل السيرفر على رابط مستقل مثل `wss://my-game-server.com`، قم بفتح التعديل على الملف `.env` (أو إعدادات Environment Variables في AI Studio) وأضف الرابط:

```env
VITE_BACKEND_URL=wss://my-game-server.com/game-socket
```

عندها سيقوم التطبيق والـ Android بالاتصال بسيرفرك فوراً بدلاً من الاتصال بالسيرفر الافتراضي الموجود في الكود.
