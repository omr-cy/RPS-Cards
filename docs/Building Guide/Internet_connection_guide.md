# دليل ربط السيرفر المحلي بالإنترنت (Localhost to Public)

هذا الدليل يشرح كيفية تحويل سيرفر يعمل محلياً على جهازك (مثل Node.js أو غيره) إلى سيرفر مرئي يمكن لأي شخص في العالم الاتصال به عبر الإنترنت. هذه الطريقة مفيدة جداً لاختبار ألعاب الـ Multiplayer أو التطبيقات قبل استئجار استضافة حقيقية.

## الفكرة باختصار
جهازك يمتلك **IP محلي (Local IP)** لا يراه إلا من هو معك على نفس الراوتر. الراوتر يمتلك **IP خارجي (Public IP)** يراه العالم.
نحن نحتاج أن نقول للراوتر: "أي شخص في العالم يطلبك من بورت 3000، أدخله فوراً إلى جهازي الداخلي على بورت 3000". وهذا ما يسمى بـ **Port Forwarding**. وبما أن الـ Public IP يتغير كلما أعدت تشغيل الراوتر، سنستخدم خدمة مثل **DuckDNS** لتربط الـ IP بدومين ثابت (مثال: `mygame.duckdns.org`).

---

## الخطوات العملية

### 1- تشغيل السيرفر محلياً
تأكد أن السيرفر الخاص بك يعمل ومفتوح ويستمع على IP `0.0.0.0` وليس `localhost`.
```typescript
// مثال في Node.js / Express
server.listen(3000, '0.0.0.0', () => {
    console.log("Server running on port 3000");
});
```

### 2- معرفة الـ Local IP الخاص بجهازك
افتح التيرمينال واكتب:
```bash
ip a
# أو
ifconfig
```
ابحث عن رقم يشبه `192.168.1.X` (مثلاً `192.168.1.2`). هذا هو عنوان جهازك في الشبكة المنزلية.

### 3- فتح البورت في جدار الحماية للحاسوب (Firewall)
أنظمة لينكس تميل لغلق البورتات للحماية. لذلك إن كنت تستخدم (Parrot OS, Ubuntu, Debian)، يجب السماح للبورت:
```bash
# إذا كنت تستخدم UFW
sudo ufw allow 3000/tcp

# إذا كنت تستخدم iptables
sudo iptables -A INPUT -p tcp --dport 3000 -j ACCEPT
```

### 4- إعداد الـ Port Forwarding في الراوتر
1. ادخل إلى إعدادات الراوتر عبر المتصفح (غالباً `192.168.1.1`).
2. ابحث عن **Port Forwarding** أو **Virtual Servers** أو **NAT Forwarding**.
3. أضف قاعدة جديدة (Rule):
   - **Internal IP**: الـ IP الذي وجدناه في الخطوة الثانية (مثال `192.168.1.2`).
   - **Internal Port**: 3000
   - **External Port**: 3000
   - **Protocol**: TCP / All
4. احفظ الإعدادات.

### 5- إعداد الدومين الثابت (DuckDNS)
لأن الـ IP الخارجي الخاص بك يتغير، استخدم DuckDNS:
1. اصنع حساباً في [DuckDNS.org](https://www.duckdns.org/).
2. قم بإضافة نطاق جديد (مثال: `mygame.duckdns.org`).
3. اضغط على أداة "Update IP" لربط الدومين بالـ Public IP الحالي الخاص بك. (يمكن تشغيل سكريبت من DuckDNS لتحديثه تلقائياً).

### 6- التوصيل في جزء الـ Client (تطبيق أو لعبة)
الآن يمكنك ضبط المسارات في ملف واحد مركزي هو `src/config.json`:
```json
{
  "ONLINE_SERVER_URL": "ws://mygame.duckdns.org:3000/game-socket",
  "ONLINE_API_BASE_URL": "http://mygame.duckdns.org:3000"
}
```
سيقوم التطبيق تلقائياً بقراءة هذه الإعدادات للاتصال بـ WebSocket والـ API الخاص ببيانات البروفايل.

### 💡 هيكلية المشروع (Project Structure)
لضمان الأمان وعدم تسريب كود السيرفر داخل تطبيق الأندرويد، تم فصل الكود تماماً:
- **مجلد `src/`**: يحتوي فقط على كود الفرونت-اند (الواجهة) الذي يتم بناؤه للأندرويد.
- **مجلد `server/`**: يحتوي على كود الباك-اند (السيرفر) وهو معزول تماماً عن عملية البناء (Build) الخاصة بـ Capacitor، مما يضمن أن الـ APK سيكون خفيفاً وآمناً.

## 🛑 ملاحظات مهمة وتخطي العقبات

### 1. مشكلة الـ SSL (Mixed Content)
إذا كان التطبيق الخاص بك عبارة عن موقع ويب (Website) ويعمل على `https://`، فالمتصفحات (Chrome, Safari, Firefox) **سترفض** الاتصال بسيرفر الألعاب الخاص بك لأنه يستخدم `ws://` بدلاً من `wss://` المشفّر.
* **الحل 1 (للألعاب والمشاريع القابلة للتحميل):** تحويل اللعبة لتطبيق أندرويد (متل ما فعلنا عبر Capacitor). كود Java / كود الـ Native في الأندرويد لا يخضع لقيود المتصفح ولا يرفض الـ `ws://`. يمكنك ببساطة عمل ميزة بـ Capacitor لإجراء الـ WebSocket في الـ Native وتمرير الرسائل لـ React.
* **الحل 2:** عمل شهادة SSL ووضع السيرفر وراء Reverse Proxy مثل Nginx.

### 2. مشكلة الـ CGNAT (Carrier-Grade NAT)
في بعض البلدان أو شبكات 4G، مزود الإنترنت يضعك خلف راوتر مركزي (لا يعطيك Public IP حقيقي).
* **كيف تعرف؟** إذا قمت بفتح البورت وكل شيء صحيح ورغم ذلك الاتصال يعطي (Connection Refused/Timeout).
* **الحل وقتها:** استخدام خدمات الأنفاق (Tunnels) مثل **Ngrok** أو **Cloudflare Tunnels** أو نقل السيرفر ليتم استضافته على خوادم سحابية خارجية (مثل Render, Railway, أو VPS).

### 3. اختبار نجاح الـ Port Forwarding بأمان
للتأكد أن الراوتر مرر البورت بنجاح قبل تشغيل التطبيق، قم بتشغيل السيرفر، ثم افصل هاتفك عن الواي فاي (استخدم 4G)، وحاول الدخول للرابط التالي من متصفح الهاتف:
`http://mygame.duckdns.org:3000` (ويفضل أن تكون قد برمجت مسار كـ `/api/health` ليرد بـ JSON).
إذا فتح الهاتف الصفحة بنجاح، فالبورت مفتوح تماماً واللعبة ستتصل دون أخطاء.
